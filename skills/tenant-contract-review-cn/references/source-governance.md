# 来源治理与城市覆盖

本包把每个来源拆分为四类，且不允许互相替代：

| 类别 | 可支持的输出 | 不可支持的输出 |
| --- | --- | --- |
| `legal_basis` | 已核实、仍有效且适用范围完整的法律信息 | 个案结论、未经核验的城市规则 |
| `structural_reference` | 条款结构、建议写法和偏离提示 | 违法、无效或当然可执行的结论 |
| `red_line` | 预定义高严重度条件及其最低证据 | 来自单一社区经历的断言 |
| `candidate_scenario` | 待核实的风险提问和补证方向 | 法律依据、红线或对个人／企业的事实指控 |

## 采纳规则

1. 法律依据只可来自最终官方发布页。记录必须含唯一 ID、效力类别、发布者、最终域名、适用地域／时间／主体／事项、生效失效日期、复核日期、状态和支持的风险命题。
2. 转载页、搜索摘要、短链、开放重定向和未确认最终地址均不得作为法律依据。状态为 `expired`、`withdrawn`、`conflicting`，或适用维度缺失时，只能触发“暂停并核实”的降级。
3. 国家层规则是所有城市的底座。北京、上海、广州分别有本包已复核的地方覆盖；深圳及其他城市必须显示 `coverage_gap`，并仅以全国规则作底座，直到补入经核验的现行官方地方来源。
4. 示范文本仅为结构参考。合同没有采用其措辞时，输出只能说明结构差异或建议写法，不得自动推定违法。
5. 红线必须指向本包 `legal_basis` 中的有效来源和最小证据要求。社区内容、个人黑名单、未经证实的企业指控、或长段原文不得进入红线。

## 社区候选与实时检索

社区候选场景必须独立保存于 `community-scenarios.yaml`；它们只可提示问题、证据缺口或待核实机制。每条记录须有来源日期、独立来源、机制、官方佐证状态、复核人和过期规则。不得把合同全文、姓名、详细地址、账号、联系方式、身份证明、对方聊天或未证实指控写入该文件。

实时检索只允许最小化、通用的法规查询，例如“广州市 住房租赁 备案 现行 规定”。查询不得包含姓名、地址、账号、合同原句、文件指纹或任何案件数据；命中上述内容时停止查询并改用用户去标识化的问题摘要。

## 复核与降级

每次使用法律依据前，校验状态、复核日期、最终官方域名及全部适用维度。若本地记录已超过其 `review_due_on`、页面有撤回／修订／冲突迹象，或案件超出地域、时间、主体或事项范围，停止确定性法律表述，记录原因并提示从官方来源重新核验。此包不替代律师的个案意见。
