# 谈判辅助契约

本页只把已经通过材料、证据、规则和决策守卫的发现转成可审查的谈判辅助。它不替代律师意见，不授权任何一方，也不得把口头承诺当作已经解决的风险。

## 单项谈判卡

每个仍可通过书面方式降低风险的发现都必须有一张谈判卡，并与 `finding_id`、材料版本、证据 ID 和规则版本相连。卡片至少包含：

```text
finding_id, risk, compatible_clause_text, non_threatening_rationale,
linked_clause_ids, linked_remedy_ids, annex_ids, document_priority_terms,
written_evidence_required, dependency_check, conflict_status
```

- `compatible_clause_text` 必须是可写入主合同、补充协议或可保存书面确认的具体中文文本，并与该发现的风险、适用文件和已知事实相容；不能用空泛的“请修改”代替。
- `non_threatening_rationale` 只说明双方可核验的履行、费用、交接或争议预防需要；不得使用威胁、羞辱、虚假法律后果或“必然违法”的表述。
- 先核验定义、交叉引用条款、违约/补救条款、附件和文件优先级约定。任何一项未知、矛盾或会改变另一文本含义时，`conflict_status` 必须为 `coordinated_edit_required`，且该卡的 `direct_paste_prohibited` 必须为 `true`。此时只可输出“联动修订清单”，不能把单段文字直接建议粘贴。
- 书面补救在进入终稿复核前必须能定位到已提交版本；聊天、电话或当面说法本身不是已落实文本。

## 分层话术与辅导

默认输出一条简短、可转发的中性请求和上述谈判卡。只有发现优先级为 `high`，或用户明确选择该事项时，才可输出完整辅导（开场、回应分支、让步边界和回合建议）。其它事项不得展开为偏好驱动的强势谈判脚本。

若要提出任何以个人偏好换取条件的让步、加价、提前付款、缩短保障期限或接受额外风险，必须先由用户逐项确认：

```text
urgency, total_cost, alternatives, paid_amounts, risk_tolerance
```

五项均为当前案件、当前材料版本下的明确确认才可创建偏好型让步；缺一项时只说明需要确认，不生成让步建议。此确认不改变证据真实性、规则适用性或四档结论。

## 对方身份、联系人与升级

联系人和升级路径必须匹配已核实身份：

| 已核实对方身份 | 首要书面联系人 | 不能据此承诺的事项 |
| --- | --- | --- |
| 房东 | 合同中可核验的出租方 | 其一定拥有处分/出租权或收款权 |
| 代理人 | 合同列明代理人，并抄送/回到被代理主体的可核验渠道 | 代理范围、期限、签约或收款授权已当然成立 |
| 经纪/中介 | 经纪机构及其经办人；涉及合同义务时回到出租/承租主体 | 经纪人可代替出租方作出合同承诺 |
| 公司 | 合同主体公司的可核验合同/授权渠道 | 联系人或盖章人已获签约、收款或处分授权 |

身份、授权、收款或费用依据未知时，谈判卡只能要求最小书面凭证并把案件保留在“暂停签约并核实”路径。发送对象、升级对象或回复渠道都不是授权已经成立的证据，也不是处理终点。

## 回覆、拒绝和口头承诺

对方拒绝、未给可接受书面补救，或仅作口头承诺时：保留风险为未解决，删除依赖该解决状态的决策检查点，回到 `Scanned`，并重新计算四档结论。只有写入合同、补充协议或可保存书面渠道并进入新的材料版本后，才可重新走终稿复核。
