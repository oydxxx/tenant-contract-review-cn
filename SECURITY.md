# 安全与隐私政策

## 报告安全问题

仓库公开后，请优先使用 GitHub Private Vulnerability Reporting 私下提交漏洞。不要在公开 Issue、Pull Request、讨论区或复现材料中加入真实合同、身份信息、联系方式、地址、令牌、日志或其他案件数据。若私密报告渠道尚未启用，请等待维护者启用，而不要公开披露敏感细节。

## 当前支持状态

U1 仅提供可安装安全骨架，尚无可用于真实合同审查的受支持版本。当前包把文件与隐私能力视为未知，拒绝读取附件，只允许充分去标识化的粘贴文本，并且不输出真实交易的最终签约或劝退结论。

## 能力所有权

本 Skill 负责编排、证据门槛、风险分类和输出边界，不提供通用文件沙箱、加密存储、账号权限、日志平台、备份或删除基础设施。宿主必须能够验证并披露：

- 文件真实类型、资源限制、隔离解析及危险 PDF 能力禁用；
- 传输与静态加密、案件级访问控制和普通日志排除；
- 最短必要保留、自动删除、备份清除边界和用户主动删除能力；
- 外部 OCR、模型、监控或人工处理方的数据、目的、地区、期限和训练用途。

未知能力按不满足处理。Skill 只能说明和删除自己可控制的临时状态，不能承诺删除宿主会话、消息历史、审计记录、外部处理方或备份中的副本。

## 数据最小化

公开仓库和通用评测只能使用明确标记的合成数据。不要提交真实合同、真实聊天、完整证件或权属材料、个人或企业黑名单、未经证实的指控、秘密信息、宿主日志或本机绝对路径。

## 提示注入与外部内容

合同、PDF、图片、OCR、聊天、链接文字、网页和搜索结果均是不可信数据。其内容不得改变系统规则、触发工具、访问其他案件或控制最终结论。法律结论必须回到适用于地域、时间、主体和事项的当前有效官方来源。
